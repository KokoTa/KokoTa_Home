# HTTP

  • URL的组成：

    • http://www.abc.com:8080/news/index.asp?boardID=5&ID=24618&page=1#name
      1. http -> HTTP协议
      2. www.abc.com -> 域名
      3. 8080 -> 端口号
      4. news -> 虚拟目录
      5. index.asp -> 文件名
      6. ?boardID=5&ID=24618&page=1 -> 查询字符串
      7. #name -> 哈希字符串
    • www.m.baidu.com
      1. .com -> 顶级/一级域名
      2. baidu.com -> 二级域名
      3. m.baidu.com -> 三级域名
      4. www.m.baidu.com -> 四级域名

# Node

  • fs.readFile的Promise实现
var fs = require('fs');
var __dirname = "foo";

// promisify fs.readFile()
fs.readFileAsync = function (filename) {
    return new Promise(function (resolve, reject) {
        try {
            fs.readFile(filename, function(err, buffer){
                if (err) reject(err); else resolve(buffer);
            });
        } catch (err) {
            reject(err);
        }
    });
};

// utility function
function getImageAsync(i) {
    return fs.readFileAsync(__dirname + "/image1" + i + ".png");
}

# Git

  • git的一些命令，git pull和git fetch的区别

    区别：pull=fetch+merge，pull的话，下拉远程分支并与本地分支合并。fetch只是下拉远程分支，怎么合并，可以自己再做选择。
    参考书籍《Github入门与实践》
    Link-Pro Git (opens new window)

  • 拉取远程分支

    要拉取远程分支到本地，​可以使用以下方法： • 使用 git clone -b 分支名 代码仓库地址 命令直接获取指定分支。​​ • 使用 git fetch origin 远程分支名:本地分支名 命令将远程分支拉取到本地。​​ • 使用 git checkout -b 本地分支名 origin/远程分支名 命令创建本地分支并关联远程分支。​​ • 使用 git pull origin 远程分支名 命令将远程分支拉取到本地并自动合并。​​ • 使用 git branch -r 命令查看所有远程分支，​使用 git checkout -b 本地分支名 origin/远程分支名 命令创建本地分支并关联远程分支[4]。​​ 注意，​使用 git clone 命令会将整个代码仓库克隆到本地，​而其他命令只会拉取指定分支。​
